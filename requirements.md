# Требования для работы проекта ATM Monitor

Для полноценного запуска и стабильной работы системы мониторинга и оптимизации инкассации банкоматов (ATM Monitor) требуется установить следующие компоненты:

---

## 1. Среда выполнения (Python)
Проект разработан на базе Python 3.10+. Выберите один из способов в зависимости от прав доступа на компьютере:

* **Для компьютеров с правами администратора (рекомендуется):**
  * Скачайте официальный установщик: [Python 3.10.11 (64-bit)](https://www.python.org/ftp/python/3.10.11/python-3.10.11-amd64.exe)
  * При установке обязательно отметьте галочку **«Add Python to PATH»** (Добавить Python в PATH).
  
* **Для компьютеров БЕЗ прав администратора (корпоративные ПК):**
  * Скрипт `start_new_pc.bat` скачает портативную версию автоматически.
  * Если вы хотите скачать её вручную, используйте архив: [Python 3.10.11 Portable (64-bit)](https://www.python.org/ftp/python/3.10.11/python-3.10.11-embed-amd64.zip)

---

## 2. Система контроля версий (Git)
Для получения обновлений и отправки ваших изменений в репозиторий:
* Скачайте установщик для вашей ОС:
  * **Windows:** [Git for Windows (64-bit)](https://github.com/git-for-windows/git/releases/download/v2.41.0.windows.1/Git-2.41.0-64-bit.exe)
  * **macOS:** Установите через Homebrew (`brew install git`) или скачайте с [официального сайта](https://git-scm.com/download/mac).

---

## 3. Зависимости и библиотеки (Python Packages)
Все необходимые библиотеки перечислены в файле `requirements.txt`. Они устанавливаются автоматически скриптами запуска (`run.sh` / `start_new_pc.bat`), но их также можно установить вручную следующей командой:

```bash
pip install -r requirements.txt
```

### Основные библиотеки:
1. **fastapi** — современный веб-фреймворк для создания API.
2. **uvicorn** — высокопроизводительный ASGI веб-сервер.
3. **websockets** — обмен данными в реальном времени (симуляция движения банкоматов).
4. **pandas** — интеллектуальная обработка табличных реестров банкоматов.
5. **openpyxl** — чтение и парсинг XLSX/Excel файлов.
6. **numpy** — математические операции и числовые вычисления.
7. **pydantic** — валидация входных данных.
8. **python-multipart** — обработка загрузок файлов в API.

---

## 4. Веб-браузер и Сеть
Интерфейс системы разработан в виде адаптивного веб-приложения:
* **Браузер:** Любой современный браузер с поддержкой HTML5 и JavaScript (Chrome, Edge, Safari, Firefox, Opera).
* **Сеть (Интернет):** Для отображения карты используется библиотека Leaflet.js с тайлами CartoDB. Для корректного рендеринга карты и иконок требуется доступ к следующим CDN:
  * `https://unpkg.com/` (загрузка иконок Lucide и стилей Leaflet)
  * `https://basemaps.cartocdn.com/` (рендеринг подложки карты)
  * `https://cdn.jsdelivr.net/` (вспомогательные JS скрипты)

---

## 5. Входные данные (Реестр XLSX)
Для работы системы необходим реестр банкоматов. Вы можете подготовить и импортировать XLSX-файл через страницу импорта (`http://localhost:8000/dashboard/import.html`).

Колонки в XLSX-файле могут называться на узбекском, русском или английском языках (система автоматически сопоставит их). Рекомендуется наличие следующих полей:
* **Номер банкомата** (atm_number, №)
* **Название / Адрес** (address, name)
* **Координаты** (широта и долгота: lat, lon, latitude, longitude)
* **Код филиала / Код области** (branch_code, region)
* **Состояние кассет** (лимиты и остатки по купюрам: 100 000, 200 000 и т.д.)
