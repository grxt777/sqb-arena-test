# 🏧 ATM Cash Intelligence — Система прогнозирования остатков и оптимизации инкассации

> **Хакатон: Тема №6** — Прогнозирование остатков в банкоматах (ATM)  
> Предотвращение cash-out и оптимизация маршрутов инкассации средствами Machine Learning

---

## 🎯 Проблема

Банки ежедневно тратят десятки миллионов на инкассацию ATM по устаревшей схеме:

- Пополнение по историческому среднему — без учёта праздников, зарплатных дней и сезонности
- Реактивный подход: банкомат уже пуст, клиент уже недоволен
- Маршруты инкассации не оптимизированы — машины проезжают лишние километры

**Результат:** cash-out, потеря лояльности клиентов, заморозка оборотного капитала в ненужных точках.

---

## 💡 Решение

**ATM Cash Intelligence** — интеллектуальная платформа мониторинга и прогнозирования наличности в банкоматах.  
Система предсказывает баланс на 24 часа вперёд, определяет риск cash-out и строит оптимальный маршрут инкассации.

### Что система делает:

| Возможность | Описание |
|---|---|
| 📊 **Live-мониторинг** | Реальное состояние 45 ATM по WebSocket, обновление каждые 2 секунды |
| 🤖 **ML-прогноз** | Предсказание баланса на 24ч и вероятности cash-out для каждого ATM |
| 🚨 **Умные алерты** | Приоритизация по риску: `HIGH` / `MEDIUM` / `LOW` |
| 🗺️ **Маршруты инкассации** | Оптимизация с 2-opt алгоритмом + интеграция с OSRM |
| 💰 **Baseline-сравнение** | Сравнение ML-подхода со старым банковским методом: сколько денег экономим |
| 📈 **Операционная аналитика** | KPI сети за 12ч / 24ч / 7 дней / 30 дней |

## 🏆 Ключевые результаты модели

| Метрика | Значение |
|---|---|
| Регрессия MAE | **27.2 млн сум** (~6.8% MAPE) |
| Регрессия RMSE | 51.3 млн сум |
| Classifier ROC-AUC | **0.9903** |
| Classifier AUC-PR | **0.9410** |
| Горизонт прогноза | 24 часа (12 шагов × 2ч) |

> На тестовой выборке: ROC-AUC = **0.987**, AUC-PR = **0.918** — модель уверенно обнаруживает риски cash-out.

### Экономический эффект

- ML-подход снижает излишнюю загрузку наличными на **сотни миллионов сум** относительно банковского baseline
- Оборотный капитал, ранее замороженный в ATM-сети, возвращается в распоряжение банка
- Инкассация приоритизируется по реальному ML-риску, а не по расписанию

## 🧠 ML-подход

### Модели

**XGBRegressor** — предсказывает абсолютный баланс ATM через 24 часа  
**XGBClassifier** — предсказывает вероятность cash-out (остаток < 20% ёмкости)

### Feature Engineering

Для каждого ATM в момент инференса формируется вектор признаков:

- **Лаги**: баланс на -1, -3, -6, -12, -24 периода (шаг = 2ч)
- **Скользящие средние**: rolling mean за 6, 12, 24 периода
- **Burn rate**: скорость расхода за последние 6 периодов
- **Time to empty**: прогноз часов до нуля при текущем темпе
- **Инкассации**: количество за последние 12/24 периода
- **Календарь УЗ**: зарплатные дни (10-е и 25-е ±2 дня), праздники, сезонный фактор
- **Профиль ATM**: кодирование типа локации (рынок, метро, жилой район, ...)
- **Часовые и дневные паттерны**: час дня, день недели, выходной

### Почему XGBoost, а не LSTM / Prophet?

| Критерий | XGBoost | LSTM | Prophet |
|---|---|---|---|
| Качество на данных | ✅ 99% AUC | хуже при коротких рядах | хуже без тюнинга |
| Скорость инференса | ✅ < 1 мс | > 100 мс | > 50 мс |
| Интерпретируемость | ✅ feature importance | ❌ чёрный ящик | частично |
| Поддержка в проде | ✅ joblib, 1 файл | требует GPU/TF | отдельный стек |

> XGBoost выиграл по метрикам, скорости и простоте развёртывания. LSTM и Prophet проверены, но уступают на наших данных.

## 🗺️ Оптимизация маршрутов инкассации

1. **Фильтрация** ATM по статусу (`critical` / `warning`) и ML-риску (`HIGH` / `MEDIUM`)
2. **Приоритетный score** = f(баланс %, ML-вероятность, часы до опустошения)
3. **Кластеризация** по количеству машин инкассации
4. **2-opt TSP** — локальная оптимизация маршрута внутри кластера
5. **OSRM** (опционально) — реальные дорожные дистанции и геометрия

**Результат**: экономия на каждом маршруте относительно round-robin baseline, меньше холостых пробегов.

## 🚀 Быстрый старт

### Требования

- Python 3.10+
- pip

### Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone <repo-url>
cd ATM

# 2. Установить зависимости
pip install -r requirements.txt

# 3. (Опционально) Переобучить модели
python train_model.py

# 4. Запустить сервер
uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload
```

```bat
start.bat
```

## 🌍 Особенности для Узбекистана

- **Зарплатный календарь**: учёт пиков спроса 10-го и 25-го числа каждого месяца (±2 дня)
- **Государственные праздники**: Навруз, День независимости, Курбан-хайит и другие — вшиты в модель
- **Сезонность**: коэффициент спроса по кварталам
- **Профили ATM**: рынки, метро, жилые кварталы, офисы — разные паттерны спроса
